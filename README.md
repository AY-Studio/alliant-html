# Alliant AirFinance Website

A modern aviation finance website built with Bootstrap 5 SCSS, Panini templating, Swup page transitions, and AOS scroll animations.

## Features

- **Bootstrap 5.3.2** - Built entirely with Bootstrap's SCSS version
- **Panini Templating** - Static site generator for reusable components
- **Swup Page Transitions** - Smooth fade transitions between pages
- **AOS Scroll Animations** - Fade-in animations on scroll (once per session)
- **Responsive Design** - Fully responsive for desktop, tablet, and mobile
- **Section Theme Helpers** - Swap between grey (`group-grey`) and navy (`group-navy`) section palettes without touching markup
- **Custom Colors** - SCSS variables to match brand colors (#223a69)
- **Google Fonts** - Mulish font family
- **Gravity Forms** - Styled contact form with validation
- **WordPress WYSIWYG** - Content styling for articles
- **Clean Code** - Well-commented SCSS and HTML

## Setup Instructions

### 1. Install Dependencies

```bash
npm install
```

This installs:
- Bootstrap 5.3.2
- Sass compiler
- Panini (static site generator)
- Swup (page transitions)
- AOS (scroll animations)
- npm-run-all (parallel script runner)

### 2. Build the Site

**Development build (Panini + SCSS):**
```bash
npm run build:dev
```

This compiles:
- Panini templates (`src/` → `*.html`)
- SCSS to uncompressed CSS

**Production build:**
```bash
npm run build
```

Compiles with compressed CSS.

**Watch mode (auto-compile on save):**
```bash
npm run watch
```

Watches both Panini templates and SCSS files.

#### Script reference

- `npm run build` – production build (`build:panini` + `build:scss`).
- `npm run build:dev` – development build with readable CSS and in-place HTML output.
- `npm run watch:scss` – watch and recompile only `scss/` → `css/style.css` via Sass.
- `npm run watch:panini` – watch Panini sources and rebuild pages.
- `npm run watch` – run both watchers in parallel.
- `npm run server` – start BrowserSync on port 8000 and watch HTML/CSS/JS for reloads.
- `npm run dev` – build once, then run both watchers and BrowserSync together.

### 3. Run Local Server

**Important:** Use a local server (not `file://`) for Swup page transitions to work:

**Option 1: Browser-Sync (Recommended - with live reload):**
```bash
npm run server
```

**Option 2: Full development mode (build + watch + server):**
```bash
npm run dev
```
This will automatically:
- Build the project
- Watch for file changes
- Start a local server with live reload

**Option 3: Simple Python server:**
```bash
python3 -m http.server 8000
```

Then visit `http://localhost:8000`

## Page Transitions (Swup)

Swup provides smooth fade transitions between pages:

- **Container:** `#swup` wraps all page content
- **Animation:** 300ms fade with navy background
- **Configuration:** `js/main.js`
- **Hooks:** AOS refresh on page view

### Customizing Transitions

Edit `scss/style.scss`:

```scss
.transition-fade {
  transition: opacity 0.3s ease-in-out;
}

html.is-animating .transition-fade {
  opacity: 0;
}
```

## Scroll Animations (AOS)

AOS (Animate On Scroll) animates content as you scroll:

- **Animation:** `data-aos="fade"` (fade-in only)
- **Delays:** 100ms, 200ms, 300ms for staggered effects
- **Once:** `once: true` (animations play once per session)
- **Duration:** 800ms
- **Offset:** 100px before element enters viewport

### Adding Animations

Add to any element:

```html
<div data-aos="fade" data-aos-delay="100">
  Content fades in
</div>
```

### Configuration

Edit `js/main.js`:

```javascript
AOS.init({
  duration: 800,
  easing: 'ease-in-out',
  once: true,
  offset: 100
});
```

## Panini Templating

Panini enables reusable components:

### Creating Pages

Create in `src/pages/` with frontmatter:

```html
---
title: Page Title
hero_title: Custom Hero Title
hero_description: Custom description
hide_cta: true
---
{{> hero}}
{{> content}}
```

### Using Partials

Include reusable components:

```html
{{> header}}
{{> hero}}
{{> footer}}
```

### Dynamic Content

Use variables in partials:

```html
<h1>{{#if hero_title}}{{hero_title}}{{else}}Default Title{{/if}}</h1>
```

## Customization

### Colors

Edit `scss/_variables.scss` to change colors:

```scss
$primary: #223a69;           // Main navy blue
$alliant-navy: #223a69;      // Primary navy blue
$alliant-light-gray: #f5f5f5; // Light gray backgrounds
```

The dark section uses a linear gradient:
```scss
background: linear-gradient(-62deg, #223A69 0%, #526FA1 100%);
```

### Layout

Edit `scss/_layout.scss` for section spacing, card styles, and grid layouts.

### Typography

Google Font Mulish is used. Customize in `scss/_variables.scss`:

```scss
$font-family-sans-serif: "Mulish", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
```
